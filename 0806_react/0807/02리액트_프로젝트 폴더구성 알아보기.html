02리액트_프로젝트 폴더구성 알아보기

리액트에 node.js설치시 생성되는 폴더의 종류와 내용 알아보기

  • [node_modules] : babel 및 webpack을 통해 리액트를 실행시켜주는 파일들을 모아놓은 폴더
  • [public] : babel이 변환시켜 연결시킨 index.html문서를 저장하고 있는 폴더
  • [src] : 리액트로 개발하는 모든 대부분의 파일이 저장되는 폴더
  • index.js : 리액트 진입파일
  • App.js : 콤포넌트 모음 파일
  • [build] : 리액트 프로젝트를 완성한 후 웹서버에 올리고자 하는 파일이 들어있는 폴더로서 npm run build명령어로 입력하면 생성됨.

public폴더의 index.html

index.html문서의 div id='root'는 실제 index.js로부터 렌더링된 내용이 삽입되는 영역이다.

src폴더의 index.js

App.js로부터 내용을 불러온 자식요소 콤포넌트를 삽입하기 위한 부모 콤포넌트 문서이다.

import React from 'react';

import ReactDom from 'react-dom';

리액트를 구현할 수 있는 플러그인을 연결한 내용이며, js파일을 불러오는 것이 때문에 import라는 키워드를 사용(es6문법)하였으며, 리액트에서는 자바스크립트 확장자 .js를 생략한다.

import 불러올 무언가의 이름 from '파일경로';

css파일은 import로 불러올때 확장자 '.css'를 작성한다.
import './index.css';

index.js파일의 root.render() 함수

    root.render(
      출력할 내용
    )

render : html문서 내용을 웹페이지에 렌더링한다.

document.getElementById('root')를 통해 아이디가 root인 곳에 App을 렌더링하겠다는 의미이다.

실제로 DOM으로 가상돔을 불러오겠다는 함수의 의미

여기서 'App /'는 App.js의 콤포넌트를 가져오겠다는 의미

완성된 프로젝트 빌드하여 웹배포하기

vs code 하단 cmd창에서 npm run build

build폴더가 생성되며 모든 파일이 들어가 있음(index.html, css, js, images..)

index.html 더블클릭하여 페이지 나오는지 확인

build폴더의 index.html실행시 내용이 아무것도 나오지 않은 경우
